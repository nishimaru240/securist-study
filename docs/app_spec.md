# 認証セキュアWebアプリケーション設計士 試験勉強アプリ 仕様書

## 概要

スマートフォンで利用できる試験勉強用スライドビューアー。ジムのトレッドミル利用中でも使えるよう、手動めくりを基本とし、自動進行もオプションで使える設計。

---

## 利用シーン

- ジムのトレッドミルなどマシン利用中に、スマホを立てかけて閲覧
- 両手が使えない状況でも自動進行モードで学習継続できる
- 通勤・移動中などの隙間時間にも対応

---

## 機能要件

### コア機能

| 機能 | 詳細 |
|------|------|
| スライド表示 | 1スライドに1トピック（見出し＋本文） |
| 手動ナビゲーション | 進む・戻るボタン（タップ操作） |
| 自動進行モード | ON/OFFで切替。間隔は秒単位で自分で設定可能 |
| カウントダウン表示 | 自動進行中、次のスライドまでの残り時間を表示 |
| 目次（章ジャンプ） | 章・セクション一覧から任意のスライドへジャンプ |
| 進捗記録 | 最後に読んだスライド位置をブラウザに保存（次回再開） |
| 文字サイズ調整 | 小・中・大の3段階切替 |

### 対象外（スコープ外）

- クイズ・一問一答モード
- ログイン・ユーザー管理
- 複数端末間での同期

---

## 非機能要件

- **対象デバイス**: スマートフォン（縦・横両対応）
- **オフライン**: GitHub Pages配信のため、基本はオンライン利用
- **パフォーマンス**: 初回読み込み3秒以内（軽量構成）
- **アクセシビリティ**: 大きなタップ領域（ボタン高さ 60px 以上）

---

## 技術スタック

| 要素 | 採用技術 | 理由 |
|------|----------|------|
| フロントエンド | HTML / CSS / JavaScript（素のJS） | 依存関係ゼロ、スマホで直接開ける |
| コンテンツ | Markdownファイル（JSONにパース） | 編集しやすく、AIで生成しやすい |
| ホスティング | GitHub Pages | 無料、独自ドメイン不要 |
| バックエンド | なし（静的サイト） | コスト・管理コストゼロ |

---

## コンテンツ管理

### 方針

Markdownファイルをテキストエディタまたは GitHub Web UI で直接編集する。

### ファイル構成（案）

```
content/
├── index.json              # 章の一覧・順序定義
├── 01_security_req.md      # 第1章: セキュリティ要件
├── 02_authentication.md    # 第2章: 認証
├── 03_authorization.md     # 第3章: 認可・アクセス制御
├── 04_session.md           # 第4章: セッション管理
├── 05_input.md             # 第5章: 入力処理
├── 06_output.md            # 第6章: 出力処理
├── 07_https.md             # 第7章: HTTPS・暗号化
└── 08_others.md            # 第8章: その他
```

### スライドのMarkdown記法（案）

```markdown
---
# スライドタイトル

本文テキスト。
箇条書きや**強調**も使える。

---
# 次のスライド

...
```

`---` で区切るとスライドが分かれる。

## ホスティング手順（想定）

1. GitHub リポジトリを作成（public）
2. `gh-pages` ブランチまたは `/docs` フォルダを GitHub Pages に設定
3. コンテンツ更新時は Markdown を編集してコミット → 自動反映

## 試験トピック・章構成

SecuriST 認定セキュアWebアプリケーション設計士の公式カリキュラムに基づく。
試験形式: CBT方式 30問、70%（21問）以上で合格。

### 第1章: セキュリティ要件
- セキュリティ要件の原則
- Webアプリケーションのセキュリティ要件（OWASP Top 10、PCI DSS）

### 第2章: 認証
- 認証の目的・認証を行うべき箇所
- NIST SP800-63B（AAL、Authenticatorのタイプ）
- 認証方式の種類（BASIC認証、DIGEST認証、フォームベース認証）
- パスワード管理（ハッシュ化、salt、ストレッチング）
- パスワード作成・通知・変更・リセット機能
- 秘密の質問
- エラーハンドリングとログ記録
- アカウントロック・パスワードリスト攻撃対策
- 二要素認証
- リスクベース認証

### 第3章: 認可・アクセス制御
- アクセス制御の方法と失敗例
- 限定公開URL
- OpenID Connect
- OAuth
- シングルサインオン（SSO）
- FIDO認証

### 第4章: セッション管理
- セッションIDの役割
- Cookieと属性値の設定（HttpOnly、Secure、SameSite等）
- セッションIDを利用した攻撃を防ぐ設計
- セッションタイムアウト設計
- ログアウト機能
- セッションIDの生成
- CSRF対策（トークン方式、SameSite属性）

### 第5章: 入力処理
- クライアント側の入力値チェック vs サーバー側の入力値チェック
- パラメーターの種類と取り扱い（URL/POST/hidden/Cookie/ヘッダー）
- 文字種・文字長の検証
- 文字エンコーディングの統一
- ファイル処理における入力
- XMLファイルの処理
- デシリアライズ対策

### 第6章: 出力処理
- HTMLエスケープ処理（XSS対策）
- SQL文の安全な組み立て（SQLインジェクション対策）
- JSON生成
- OSコマンド呼び出し対策
- HTTPレスポンスヘッダーの管理
- リダイレクタを使う際の注意事項（オープンリダイレクト対策）

### 第7章: HTTPS・暗号化
- SSL/TLSの仕組み
- 証明書の検証とフィッシング対策
- 証明書に対する攻撃（CA侵害、偽発行、中間CA、ダウングレード）
- 安全な暗号アルゴリズムの選定
- プロトコルバージョンの選定
- フィッシングサイト対抗策（DMARC・SPF・DKIM、HSTSプリロード、CT監視）

### 第8章: その他
- エラーメッセージのハンドリング
- ログ記録・ユーザーへの通知
- 暗号アルゴリズムと乱数（CSPRNG）
- CORS（Access-Control-Allow-Origin）
- CAPTCHA・レート制限
- クリックジャッキング対策
- キャッシュ制御
- 言語・フレームワーク・ライブラリの選定
- 言語環境のセキュリティ設定
- 開発プロセスやパイプラインのセキュリティ（Secure SDLC、シークレット管理）
- 用意すべきドキュメント
